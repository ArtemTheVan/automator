== Платформа и ограничения

* Целевая ОС — *Windows 10* и новее (`_WIN32_WINNT=0x0A00`).
* Сборка ориентирована на MinGW-w64 в среде MSYS2 (`mingw64`); MSVC не тестировался.
  В Makefile-ах используются POSIX-утилиты (`rm`, `mkdir -p`, `cp`) — они доступны
  как в MSYS2, так и в Git Bash.
* Функции OpenCV/OCR опциональны; при их отсутствии библиотека собирается без них и
  предоставляет только эмуляцию ввода и захват экрана. OpenCV-зависимый код в
  `opencv_wrapper.cpp` спрятан под `#ifdef HAVE_OPENCV`, при отсутствии библиотеки
  компилируются stub-функции.
* `simulate_keystroke` принимает UTF-8 и отправляет символы через `KEYEVENTF_UNICODE`,
  поэтому работает с любой раскладкой (включая кириллицу и эмодзи).
* Эмуляция мыши использует виртуальный экран (`MOUSEEVENTF_VIRTUALDESK`) и корректно
  работает на нескольких мониторах.
* OCR вызывает `tesseract.exe` через `CreateProcess` (без `system()`/shell). Путь к
  `tesseract.exe` задаётся явно через `ocr_set_tesseract_path()` или переменную
  окружения `TESSERACT_EXE`; иначе ищется в стандартных местах установки.
  Будущая работа: миграция на C API (libtesseract) для устранения накладных расходов
  на запуск процесса.
* GUI собирается со статической линковкой `libgcc`/`libstdc++` для упрощения дистрибуции.

=== Безопасность

* Команды Tesseract формируются с экранированием аргументов и запускаются через
  `CreateProcess` с массивом параметров — защита от command injection.
* Поиск `libautomator.dll` намеренно не включает `%TEMP%` (защита от DLL hijacking);
  явный путь задаётся через `AUTOMATOR_DLL_PATH`.
* Все строковые операции в C-коде используют `snprintf` и явное копирование с
  ограничением длины.


== Сборка документа

Документ написан в формате AsciiDoc, разбит на тематические файлы и подключается через
директивы `include::`. Диаграммы PlantUML рендерятся расширением `asciidoctor-diagram`.

[source,bash]
----
# Установить инструменты
gem install asciidoctor asciidoctor-diagram

# Сгенерировать единый HTML с диаграммами
asciidoctor -r asciidoctor-diagram docs/project.adoc

# Или PDF
asciidoctor-pdf -r asciidoctor-diagram docs/project.adoc
----

Для рендеринга PlantUML требуется установленный `plantuml.jar` или сетевой сервис
PlantUML.

При необходимости каждый файл `NN-*.adoc` можно собрать отдельно, добавив в его начало
заголовок документа и нужные атрибуты:

[source,asciidoc]
----
= Заголовок раздела
:toc:
:source-highlighter: rouge
:plantuml-format: svg

include::02-quickstart.adoc[]
----
